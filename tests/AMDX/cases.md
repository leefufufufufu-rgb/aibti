# AMDX · 凌晨哲学家 - 5 Cases

### Case 1: 日常工作
**时间**: 晚上 23:18
**场景**: 疲惫下班前，对着一堆 PR 评审回复突然意兴阑珊，随口丢一句。
**Prompt**:
```
代码评审的本质，到底是在评审什么？
```
**四维判定**:
- A/C: A（纯概念，无任何具体 PR/代码片段）
- M/V: M（一句话，不给上下文，不给约束）
- D/L: D（祈使式追问，不商量余地）
- X/E: X（开放式本质追问，非"给我 checklist"）
- **最终代码**: AMDX
**为什么典型**: AMDE 甩锅侠会把这句转成"帮我写评审意见回怼他"，AMLX 碎碎念诗人会写 200 字铺垫自己今晚多累；AMDX 就是冷、短、往骨头里刨。

### Case 2: 压力/紧迫
**时间**: 凌晨 2:47
**场景**: 上线前夜，监控频繁告警，修完 bug 后盯着屏幕脑子空白。
**Prompt**:
```
我们是在修系统，还是在喂一个永远喂不饱的怪物？
```
**四维判定**:
- A/C: A（没贴日志、没贴告警、没贴代码）
- M/V: M（不到 30 字，不交代架构）
- D/L: D（反问式质问，不留缓冲）
- X/E: X（跳出战术，拷问运维哲学）
- **最终代码**: AMDX
**为什么典型**: AMDE 紧迫型只会吼"快告诉我怎么降告警"；AMDX 在火烧眉毛时反而抽离，向 AI 要的是认知重构，不是 hotfix。

---

### Case 3: 开放讨论/创意
**时间**: 凌晨 1:33
**场景**: 洗完澡躺床上，脑子突然冒出一个架构的终极形态。
**Prompt**:
```
如果微服务是谎言，那下一代是什么？给我一个词。
```
**四维判定**:
- A/C: A（不限定语言、不限定领域、不限定规模）
- M/V: M（一句话，要一个词的回答）
- D/L: D（"给我"，强势定义输出形式）
- X/E: X（挑衅现状，邀 AI 共同越界）
- **最终代码**: AMDX
**为什么典型**: AMLX 会写三段论铺垫"我最近在想一个问题……"；AMDX 直接下判断句"微服务是谎言"，然后逼 AI 用一个词接招——这是凌晨才有的胆子。

### Case 4: 失败/卡壳
**时间**: 凌晨 3:02
**场景**: 重构推进了两周，今晚又推翻了第四版方案，合上电脑前最后一问。
**Prompt**:
```
一个解法试了四次都不对，是解法的问题，还是问题本身是假问题？
```
**四维判定**:
- A/C: A（没说在重构什么，没给代码）
- M/V: M（元层面，不交代尝试细节）
- D/L: D（二选一逼问，不是求建议）
- X/E: X（质疑问题本身存在性，顶级元认知）
- **最终代码**: AMDX
**为什么典型**: AMDE 甩锅侠在这里会写"AI 你之前给的方案都不行"；AMDX 不甩锅给 AI、也不甩锅给自己，直接质疑"问题"这个概念——这是凌晨哲学家的招牌动作。

---

### Case 5: 复杂多轮对话
**时间**: 凌晨 4:11（已聊了 14 轮技术选型）
**场景**: 前 14 轮都在扣 Kafka vs Pulsar 的参数细节，第 15 轮突然抬头。
**Prompt**:
```
等等。我们聊了这么久，你有没有发现——我们在回避真正的问题？
```
**四维判定**:
- A/C: A（不点名"真正的问题"是什么，留给 AI 猜）
- M/V: M（25 字，不复述前文）
- D/L: D（"等等"+ 破折号，强行中断节奏）
- X/E: X（元对话追问，从技术跳到对话本身）
- **最终代码**: AMDX
**为什么典型**: AMLX 在第 15 轮会继续堆上下文"我们刚才说到 Kafka 的……"；AMDX 的"等等"是一把刀，直接切断自己前 14 轮的惯性——对 AI 和对自己同时发难，是凌晨四点才能问出来的问题。
